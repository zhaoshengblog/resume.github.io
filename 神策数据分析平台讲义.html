增长黑客

增长黑客这一概念起源于美国互联网行业，最早由 Sean Ellis 提出，已经帮助硅谷多家公司完成产品的快速增长。其中最著名的案例包括 Dropbox，Hotmail，Facebook。近年来，增长黑客的概念传到国内，也引起了众多创业者的关注。

增长黑客的核心是驱动用户飞涨

增长黑客，英文「Growth Hacking」，指的是创业型团队在数据分析基础上，利用产品或技术手段来获取自发增长的运营手段。说通俗一点，一家初创公司或者大中型公司内部针对新产品线而诞生的创业 型团队，以数据驱动营销，以市场指导产品，通过低成本的手段解决公司产品早期增长问题。

增长黑客最经典的模式是「AARRR」转化漏斗模型。围绕每一层漏斗环节，增长黑客可以采用以下几个 方式来提升下一环节的用户数量，即 Acquisition（获取用户）、Activation（激发活跃）、Retention（提高留存）、Revenue（增加收入）、Referral（传播推荐）。

整合是为了占据消费者心智，而链路是为了驱动消费者行为。

首次实现品牌人群资产定量化、链路化运营

“品牌人群资产”很重要，是大家都知道的。比如可口可乐的传奇总裁罗伯特伍德鲁夫说：即使可口可乐 全部工厂都被大火烧掉，给我三个月时间，我就能重建完整的可口可乐。

这位总裁为什么敢有如此豪言？

最重要的是可口可乐品牌有强大的消费者人群资产，那些听过可口可乐的人、喝过的人、一年买很多次 的人。

放在过去，”人群资产”是一个很难量化统计的概念。我们只能定性说可口可乐的人群资产一定比康师傅 的多，但是具体有多少是不知道的。

于是，阿里就推出了一个可以把品牌在阿里系的人群资产定量化运营的模型，这也是支撑它全域营销概 念落地的关键一环，这个模型叫做：A-I-P-L

A（Awareness），品牌认知人群。包括被品牌广告触达和品类词搜索的人；
I（Interest），品牌兴趣人群。包括广告点击、浏览品牌/店铺主页、参与品牌互动、浏览产品详 情页、品牌词搜索、领取试用、订阅/关注/入会、加购收藏的人；
P（Purchase），品牌购买人群，指购买过品牌商品的人；
L（Loyalty），品牌忠诚人群，包括复购、评论、分享的人。

品牌所有AIPL资产数据都可以被存在数据银行（Data Bank）中，靠的是用户在阿里体系那个共通的身份（UNI-ID）。

消费者旅程（CXJ）与消费者时刻（MOT）

消费者旅程（Customer Journey或Customer Experience Journey，简称CXJ，也称消费者历程）是经典的营销理论之一。

消费者旅程的核心思想———无论它的表现形式是5A（即Aware，Appeal，Ask，Act，Advocate），还 是AIPL（即Awareness，Interests，Purchase，Loyalty）———都将消费者的购物行为描述为从认知到 兴趣，再从兴趣转化为购买，以及从购买转化为忠诚的一连串先后发生的过程。这一过程被称为消费者 旅程。

消费者时刻（Moment of Truth，简称MOT）是CXJ中的一些关键”里程碑”似的节点。例如，搜索某个产品，又或者把这个产品的信息分享给其他人。这个概念最初来源于宝洁。

你可以发挥想象———CXJ的全过程，就像或明或暗若隐若现的一条弯弯曲曲的道路，而MOT就像这个道 路中间燃烧的火把，指引着这条路的方向。

我们无法直接控制CXJ，但能透过MOT来对CXJ施加影响。甚至很多时候，我们也无需参透CXJ到底是什 么，我们只是在设计MOT

在讲解访问深度之前，我们要简单了解一下Session分析的原理

步骤 1：将用户的行为序列，按照发生时间远到近进行排序步骤 2：以第一个行为作为起点，向后进行匹配。

若匹配到的是一个「开始事件」：那么会自动切断会话；以这个「开始事件」作为起点，进行第二个 session 的匹配

若匹配到的是一个「结束事件」：那么会将这个「结束事件」划入到当前会话中；以「结束事件」的下 一个事件作为起点，进行第二个 session 的匹配

若在切割时间内没有匹配到任何事件：那么就会自动切断会话；以下个事件作为起点，进行第二个 session 的匹配

  1. 跳出率： Session 中只发生一个事件的 Session 个数除以总 Session 数。比如有三个 Session，第一个 Session 事件序列为 A,B；第二个 Session 事件序列为 A；第三个 Session 事件序列为A,C,B；则 Session 总体的跳出率为 1/3。

  2. 退出率： Session 的退出率包括 Session 中某个事件的退出率 和 Session中任意事件的退出率。某个事件的退出率指该事件作为 Session 的结束事件的次数除以该事件发生次数，任意事件退出率指 Session 数除以 Session 中所有事件发生次数。比如有三个 Session，第一个 Session 事件序列为A,B；第二个 Session 事件序列为A；第三个 Session 事件序列为A,C,A；则 Session 中A事件的退出率为 2/4, 任意事件的退出率为 3/6。

  3. Session 时长： Session 内最后一个事件触发的时间减去 Session 内第一个事件触发的时间。

  4. Session 深度： Session 内触发事件的次数。

  5. Session 内事件时长： 假如某 Session 内事件触发顺序为 a > b > c > d，则事件 a 的时长为 b 减去 a，事件 d 的时长未知。

  6. Session 初始事件： Session 内第一次触发的事件。

  7. Session 属性： d 处的 Session 属性是指一个 Session 中初始事件的属性。比如一个 Session 的事件序列为 A,B,C；A 事件的操作系统为 iOS，B 事件的操作系统为 Android，C 事件的操作系统为空，则这个 Session 中的 Session 属性操作系统应该是 iOS，是第一个事件对应的操作系统属性值。

识别虚假流量的常见维度

一般来说，真实流量一般自然 （真实的流量在各个维度中表现一定是自然的）与多样（网民的喜好各不相同，行为一定也是 多样的）。而对于虚假流量，常表现出一定的目的性（虚假流量的产生一定和某个特定的目的有关）和规律性（特定的目的导致虚假流量一定有特殊的规律）。

案例1：某社交软件

新用户的注册转化率下降，之前猜测的多设备登录的场景，会造成漏斗分母变大，导致转化率下降。经 过分析发现，用户人均设备数小于1，既说明用户多设备登录影响很小，也间接说明用户会存在较多的 单设备多账号的情况。

另外，老用户会趋向于点击”登录”而不是”注册”，如果多设备登录造成分母变大，相应的点击”注册”按钮 比例应该降低，但实际上点击注册的比例是升高的，这说明来访用户更多是真实的新用户。

因此，可得出：用户不趋向多设备登录，不会对转化率的下降造成影响。

二、新用户注册转化率下降原因推测

我们通过现有的数据做了简单分析，推测以下原因会导致转化率下降：

  1. 虚假流量。排查数据发现，2月4日激增了一批注册用户，查看行为序列发现这个群体的设备反复注册，推测为虚假用户。该批用户量较大，后续会造成留存的降低。后续可通过用户分群创建规则，将所 有类似行为特征的用户筛选出来，降低对数据分析的干扰。

  2. 过年期间，iOS端数据的异常下降。

埋点方案的制定—事件设计

采集用户行为数据，首先需要根据业务分析需求明确采集的目标行为，进一步搞清楚应该在哪些地方埋 什么样的点。这个环节的输出物一般被称之为”埋点需求文档（DRD）”。在大部分互联网公司，规范的 产品迭代流程是，业务侧产品经理在输出”产品需求文档（PRD）”的同时，数据产品经理或分析师等角 色需要同步输出 DRD，双方的需求同步进入开发和测试验收。

由于神策的底层数据模型是 Event + User 的事件模型，因此埋点在神策分析里被称之为”事件”，埋点需求文档则被统称为”采集方案设计”，本节的工作需要借助神策方提供的《数据采集方案》模板来完成

采集方案思路

采集方案设计的核心思路，大体来说分为如下几点：

  1. 将用户指标拆解为单个或多个行为动作；
  2. 将需要分析的目标动作抽象为”事件”，添加事件维度；
  3. 根据业务需求，整体完善采集方案设计；

A. 设计为同一事件，适用场景：各事件所需属性相差不大；平时分析场景多整体分析。

B. 设计为不同事件，适用场景：各事件所需属性相差很大；分析场景多分别分析。如果采用本思路，也 建议在一些相同属性上用一样的属性名称，便于今后使用”虚拟事件功能”来整体分析。

魔法数字

“Facebook一周之内添加7个好友”的例子，基本都成为增长圈里的标配案例。
大家看到这个案例的时候有没有思考过？

  1. 为什么是添加好友这个行为？

  2. 为什么是在一周内？

  3. 为什么是7个好友呢？

其实，用户一切留存的本质其实都是感受到了产品的价值，让用户有留下来的理由，他才会留存。
所以回到Facebook的例子上分析，因为它是一个社交平台，没有好友如何社交？

如果没有添加好友这个行为，用户肯定不能感受到Facebook 的价值。

为什么是10天内添加？大家可以看看下面这张留存曲线图。

上面的走势图，有2个非常明显的区间，我把图中的2个区间分为：选择期和平稳期。
大家可以发现选择期过后的曲线就处于平稳阶段，所以facebook才有了10天内逻辑。
10天后属于他们产品的平稳期，但过了平稳期也意味着大部分的用户已经流失。

所以要在10天内的选择期内，让用户感受到产品的价值，也就是10天内完成“添加好友”这个行为，才能止损。

为什么会是7这个魔法数字？
这是我接下来重点讲的地方。

我们在做用户留存的时候，应该重点关注留下来的那部分人他们做了什么，而不是流失的那部分人。
因为用户流失的影响因素有很多，但是用户留存的原因，永远就只有一个：感受到了产品的价值。
Facebook就是发现，留下来的用户他们都有一个共同点，那就是都添加了7个以上的好友：

  • 当用户添加7个好友以下的时候，流失的波动就很大；
  • 当达到7个的时候，流失就非常平稳；
  • 而超过7个的时候，流失趋势和7个的时候相差不大。

因此，7就是一个分水岭。

错误的做法：

某互联网公司toB业务的销售运营，发现了一个魔法数字：只要销售们打电话100次以上，通话100分钟以上，成交率就会明显飙升（如下图），于是销售运营制定了双100任务，要求人人每天达到这两个指标，还在电话间挂上了“只要没打死，就往死里打”的大标语。结果搞得销售们叫苦连天，疯狂加班，还屁业绩没有，离职率也上去了。现在请数据分析师找找原因。

电商搜索模块数据运营策略

在电商产品中，用户往往通过搜索功能直接表达购物意愿。相比推荐位和分类导航，搜索功能更能体现用户的主观购物意愿。在大多数场景中，搜索功能也是最主要的购物流量入口。因此本文档从用户搜索关键词、点击、购买切入，评估用户对于不同商品的需求情况，不同关键词当前的运营状况是否健康，是否有用户还未满足的商品需求等。

场景应用

在大多数产品中都有搜索功能，用户往往通过搜索功能直接表达自己在产品內的诉求，起到了引导用户走向的重要作用，搜索的结果是否合理直接影响用户的体验。比如电商产品，相比推荐位和分类导航，搜索功能更能体现用户的主观购买意愿，在大多数场景中，搜索功能也是主要的流量入口。

本方案从用户在此流程中最重要的三个触点（搜索关键词、点击、关键转化）切入，评估：
1、用户对于不同商品的需求情况
2、不同关键词当前的运营状况是否健康
3、是否有用户还未满足的商品需求

价值如何分析

1.指标体系∶为了更深一步的了解数据背后的本质，进行有效的分析，我们需要准备以下相关指标

2.分析要点：多维度细分及下钻是探究本质的重要方法。
以下分析将会运用到「事件分析」、「漏斗分析」、「分布分析」三大分析模型。

A.评估用户对于不同商品的需求情况

1）搜索结果页「触发人数」&「次数」
目的：了解用户对于不同商品的需求强度
分析方法：在「事件分析」模型中査看毎日使用搜索功能的用户数和次效，了解搜索这个功能的基本使用情况。按关键词进行分组查看，通过各个关键词搜索结果页展示次数的排序，可以判断出用户对于不同商品及品类的需求强度。同时根据当前热门的搜索关键词了解当下的热点，可以及时针对热门商品做活动或者是参考搜索次数较高的品类词调整商品分类或者现有分类导航

2）关键词搜索「频次」
目的：通过搜索频次较高的关键词了解近期用户的关注点是什么。
分析方法：在「分布分析」模型中选择某个时间段，然后按搜索关键词进行分组，观察不同关键词被单个用户重复搜索的次数，从而发现那些「周期性」需求词，或者品类词，从而给到运营上一些参考，比如周期性的针对需求商品做活动可以参考用户的搜索关键词。

B.评估不同关键词当前的健康状况

1）搜索结果点击「人数」&「次数」
目的：了解搜索结果是否符合用户的需求。
分析方法：查看点击了搜索结果的人数和次数，可根据搜索关键词、点击排序位置等维度分别观察搜索结果点击情况

2）搜索结果「点击率」
目的：了解用户对搜索结果的满意度。
分析方法：搜索结果点击率=搜索结果点击次数/搜索结果页次数，在事件分析中使用自定义指标进行计算，然后按关键词进行分组，查看不同关键词的点击率，可得知当前用户需求强烈的商品或品类目前的满足情况。点击率越高，说明用户对搜索结果越满意，越有兴趣去查看商品的详情。

3）关键词搜索「结果数量」
目的：分析搜索关键词是否有精准、足够的搜索结果
分析方法：在事件分析中查看搜索结果数量总和，按关键词进行分组，观察关键词的搜索页结果数量，可得知对于目前的搜索需求，是否有足够多的商品供应去满足用户需求。理论上商品供应越多，用户越有可能找到自己所需要的商品。

4）搜索结果点击「位置均值」
目的：评估搜索结果的排序是否合理。
分析方法：查看搜索结果点击位置的均值，搜索结果点击的位置反应了搜索结果的排序是否合理，正常情况下搜索结果点击位置均值应该尽可能等于1，如果该值越大，说眀此关键词下面的结果排序越不符合用户需求，需要及时与研发沟通排序规则。（分析的时候需要结合位置均值和关键词一起来看，假如用户搜索的是品类词，那么点击的位置均值肯定不可能接近于1，通常用户在这种情况下都会点击查看比较多的商品，所以位置均值这个指标要贴合具体的场景去做参考。

C.评估是否有未被满足的商品需求

1）关键词搜索「结果数量
目的：了解哪些关键词没有充足的搜索结果
分析方法：在事件分析中查看搜索结果数量总和，按关键词进行分组，观察τoP50关键词的搜索页结果数量，如果在这些头部关键词中岀现了搜索结果数量很少，甚至为零时，应该及时分析原因。如不是因为关键词太过精准，只能匹配一件商品，那此关键词对应的商品及品类需要及时通过运营手段进行补充，比如关键词对应具体商品，则从供货端应该立即补充该类商品；如果关键词对应品类或者某种描述性话语，则可以考虑对现有商品库中的商品元信息进行修改，使得某一批商品可以与此关键词产生关联

2）关键词「点击-购买转化率」
目的：根据转化率判断用户的需求是否被真正的满足。
分析方法：在漏斗分析中按关键词进行分组，观察用户从搜索结果点击到购买的转化率，如果某关键词的点击-购买转化率岀现问题，说明即使现有的搜索结果足够吸引用户点击，但在进入详情页后，并不能真正促成下单，可能的原因是商品实际描述或者用途与用户搜索的意愿并不相关。那么在这种情况下，即使该关键词点击率很高，也需要运营及时干预，优化该关键词下面的商品供应或排序

小程序裂变

由于人以类聚，利用口碑传播在大多推广场景里是效果很好的拉新渠道，体现在拉新 ROI 及新客转化质量指标上表现较好。微信作为国内最大社交网络，利用其生态进行社交裂变是当前的推广运营业务热点之一，特别是小程序出现后，更是降低了变现微信社交流量的门槛。因此，本方案重点介绍在微信生态内，借助小程序形态，如何进行社交裂变的场景埋点和数据分析，从而提升社交裂变拉新数量与效果

场景应用

社交裂变场景拆解

不管是一对一还是一对多的传播场景，都存在两类角色：传播者、接受者。社交裂变就是希望利用传播者的影响力，高效地对接受者完成特定目标转化（拉新、购买、品牌曝光等等）

社交裂变目标

根据上述的场景描述，社交裂变的业务提升目标可以分解为以下四个部分

传播者产生→如何产生更多（优质）传播者
信息传播→如何提高传播信息的打开率
接受者转化→如何提升接受者的转化效率
裂变传播→如何提高接受者的裂变传播行为

A.传播者产生
小程序的分享传播主要有两大场景，一类是在任意页面，用户都可以利用小程序自带的分享功能传播出去；另一类是产品的裂变产品，比如 Luckin cofe的邀请好友页、拼多多的砍价分享等等。从技术角度来看，两个场景的实现方式是一致的，但从分析角度来看，两者的在分析指标选取上有所差异

  • 整体小程序分享情况

    • 小程序分享触发次数、人数、人均次数
    • 分享渗透率=小程序分享人数/小程序启动人数
    • 不同页面路径下的分享渗透率

  • 传播产品分享情况

    • 传播页面访问次数、人数、人均次数
    • 传播页面渗透率=传播页面浏览人数/小程序启动人数
    • 产品分享率=传播页面分享人数/传播页面浏览人数

分享带来的核心转化行为触发人数、总体占比

  • 按启动场景、分享页面、分享层级、分享文案等维度下钻

运营类

个性化 Push 实现与效果分析

App Push 对于每一位应用开发者而言，都是不可忽视的核心运营手段之一。从产品基础服务通知到营销内容触达，Push 通知被应用于产品生态中的方方面面。但是在实际的 Push 通知效果数据监测于评估时，Push 通知的送达及点击行为往往不能与 App 内的操作行为串联，出现数据链条断裂的情况。该场景解决方案旨在将 Push 通知从送达到后续用户所产生的操作进行串联，从而实现针对 Push 效果的全面诊断与优化。

场景应用

Push作为App端运营的最常用渠道，是在用户未使用App状态下，主动推送内容或信息

  • 通知某件事发生：如支付成功的扣款提醒、发货提醒、订单超时取消提醒等
  • 通知平台动态：促进用户进入查看或参与，提升平台活跃、转化和留存，如某条资讯，某个打折活动等

本方案目的在于通过对 App Push的数据指标体系的构建和分析，帮助业务实现以下三个方面的提升

  • 提高点击率及转化效果：精准、多样、高效
  • 提高通道利用效率：合理利用通道名额，提高推送覆盖
  • 提升用户体验：降低关闭通知渠道、卸载App风险

价值如何分析

1.指标体系：为了更深一步的了解数据背后的本质，进行有效的分析，我们需要准备以下三大分类指标
关键维度：操作系统、消息内容、是否营销、Push目的。
注∶因各 Android厂商Push通道接口标准与送达规则不同，难以统一釆集，故送达相关指标只涉及ioS。

Push概览指标体系

A Push触达与转化

  • 1Push通知送达人数&次数（仅IOS）
  • 2Push通知点击人数&次数
  • 3Push点击率（仅IOS）
  • 4Push目标事件转化率（漏斗）

B Push效率

  • 1Push通知授权率
  • 2Push通知覆盖率（天，仅IOS）
  • 3Push通知盖率（周，仅IOS）

C Push体验

  • 1Push通知触达次数分布（仅IOS）
  • 2Push通知触达天数分布（仅IOS）

2.分析要点：多维度细分及下钻是探究本质的重要方法
以下分析将会运用到「事件分析」、「漏斗分析」、「分布分析」三大分析模型。

A.Push到达与点击概况
1.Push通知「送达人数」&「次数」（仅IOS）
目的：了解Push每日实际触达到多少用户数多少次数，反应渠道在触达层面发挥的实际价值。
分析方法：根据Push消息內容、是否营销类、Push目的、操作系统等维度分别观察数据差异。

2.Push通知「点击人数」&「次数」
目的：了解Push每日实际带动了多少用户、多少次的访问，从而了解渠道实际对App活跃的贡献。
分析方法：根据Push消息內容、是否营销类、Push目的、操作系统等维度分别观察数据差异。

3Push通知「点击率」（仅IOS）
目的：整体评估Pvsh触达本身的转化效率，点击率受人群选择精细化和发送内容影响较大。
分析方法：一般全局的点击转化在5个点之上，过低要引起重视，及时诊断问题。

B.Push渠道利用效率
1.Push通知「授权率」
目的：Push通知授权率是评估用户通过Push保持与App接触的意愿，也是通知到达的基本保障。
分析方法：整体Push授权打开率低的话，需要分析Push策略是否对用户产生了打扰需要调整，以及在产品适当的位置和时机去提醒用户开启Push授权

2.Push通知「覆盖率」（天，仅iOS目的：Push通知翟盖率是评估Push渠道是否被有效利用的指标，每天的日活中有多少用户被触达，同时也是侧面反馈Push为日活做贡献的价值
分析方法：覆盖率低的话，需要进一步查看，是Push本身使用的太少，还是Push的触达比例低，并针对性去找解决方案

3.Push通知「覆盖率」（周，仅iOs）
目的：从周的角度来看，Push对周活跃用户的覆盖，能进一步观察Push渠道覆盖用户的广度。
分析方法：同2，覆盖率低的话，需要进一步查看，是Push本身使用的太少，还是Push的触达比例低，并针对性去找解决方案

C.Push渠道体验评估
1.Push通知「触达次数分布」（仅ioS）
目的：Push通知的次数分布上，可以看到收到Push的人群中，每个人一天收到了多少次，太多次的Push对用户来说意味着打扰
分析方法：一般建议，一个用户每天至多收到的营销运营类消息不超过3~5条，具体的数量边界，会跟产品的类型有关系，服务工具类的产品建议少一些，资讯内容类的可以多一些。

2.Push通知「触达天数分布」（仅os）
目的：Push通知的天数分布，可以看到收到Push的人群中，一周内会收到消息的天数。
分析方法：一般建议，一个用户每周至多收到的营销运营类消息的天数在2~3天，具体的边界会跟产品的类型有关系，服务工具类的产品建议尽量少一些，资讯内容类的可以多一些

综合用例：分类或特定Push触达-转化效果诊断思路

1.Push推送「点击转化」
根据「Push点击率」判断推送后的唤起效果，一般全局推送点击率在5~10%左右，不同类型的会有差异，点击率较低的主要原因会有
人群精准性
推送的人群是否可以进一步细分，查看不同属性人群的点击率是否有比较明显的差异，若差异较大，可考虑仅推送转化率高的人群，对较低的人群考虑其它方案
推送内容的吸引力
如果整体点击量都偏低，内容吸引力的因素可能比较大，包括文案、刺激性信息等，可參考历史表现较好的push内容的特点，迭代后持绠测试优
推送时间段
可以查看下推送人群相对活跃的时间分别是什么时候，多找几个时间点进行测试，找到最优的推送时机

2关键业务「各个环节的转化率」
通过观察业务转化漏斗，了解用户点击Push后各个业务关键环节的转化效果，以及转化率低的主要原因。建议的排查、解决方案如下
推送是否定向跳转相应功能页面
Pυsh推送时设定承接页面，可有效提高用户进入App后的体验，显著提高相应的转化率。若转化率较低，可能是没有做精准跳转，或者跳转功能异常，可查看Push推送增加设置，或做技术排查
承接页面和内容是否有吸引力
比如推送一个活动、推送一个视频、推送优惠券等，这些功能服务是否有吸引力，决定了业务转化的关键点，承接页面到下一步的转化率低的话可查看承接页面的跳出率、主流程点击等交互转化，从而定向优化
功能体验是否流畅
用户有意愿做下一步交互后，相应的功能服务是否能让客户顺利的完成转化，与功能本身的设计体验有关，可详细查看功能流程的转化，以及具体页面的点击交互，找到阻塞点并定向优化
